# Лабораторна робота №6

Програма loader.c виконує наступні етапи завантаження бінарного файлу:
- Аналіз заголовка ELF Header: Зчитування структури Elf64_Ehdr для отримання адреси точки входу та зміщення таблиці програмних заголовків.
- Парсинг сегментів: Ітерація по всіх сегментах файлу. Завантажувач ігнорує службові сегменти та фокусується лише на тих, що мають тип PT_LOAD.
- Виділення пам'яті: Для кожного сегмента PT_LOAD викликається системний виклик mmap. Пам'ять виділяється за конкретною віртуальною адресою , що вказана у файлі.
- Копіювання даних: Вміст сегмента копіюється з файлу в оперативну пам'ять. Якщо розмір у пам'яті  більший за розмір у файлі , залишок заповнюється нулями.
- Передача керування: Завантажувач створює покажчик на функцію за адресою точки входу та здійснює стрибок на виконання завантаженого коду.

## Під час розробки виникли наступні проблеми, які були вирішені:
- Зміщення курсору файлу: Використання lseek всередині циклу зчитування заголовків призводило до помилок читання наступних елементів таблиці. Рішення: попереднє зчитування всієї таблиці програмних заголовків у масив перед початком мапування.
- Залежність від стандартних бібліотек: Програми, скомпільовані зі стандартною бібліотекою libc, вимагають складного налаштування стеку та оточення. Для тестування було створено мінімальний hello.c, що використовує прямі системні виклики через асемблерні вставки .
- Вирівнювання пам'яті: Реалізовано вирівнювання адрес по межі сторінки (4KB) для коректної роботи mmap з прапором MAP_FIXED.

## Компіляція завантажувача

gcc loader.c -o loader

## Підготовка тестової програми

gcc -nostdlib -static -fno-pie -no-pie hello.c -o hello

## Запуск

./loader hello
